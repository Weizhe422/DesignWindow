# DesignWindow 功能說明文件

## 概述
本文檔詳細說明 DesignWindow 應用程式中最近實作的三個核心功能：**儲存 (Save)**、**另存新檔 (Save As)** 以及 **文字編輯器 (Text Editor)**。

---

## 一、儲存功能 (Save Function)

### 1.1 功能位置
- **檔案選單**：檔案(E) → 儲存(S)
- **工具列**：儲存按鈕（磁碟圖示）
- **快捷鍵**：`Ctrl+S`

### 1.2 程式碼實作
```cpp
void MainWindow::onSaveTriggered()
{
    if (currentFilePath.isEmpty()) {
        // 如果沒有當前檔案路徑，則呼叫另存為
        onSaveAsTriggered();
    } else {
        // 儲存到當前檔案
        saveToFile(currentFilePath);
    }
}
```

#### 逐行程式碼說明：

**第 1 行：函式定義**
```cpp
void MainWindow::onSaveTriggered()
```
- `void`：函式沒有回傳值
- `MainWindow::`：這是 MainWindow 類別的成員函式
- `onSaveTriggered()`：函式名稱，當「儲存」動作被觸發時呼叫此函式（與 action_S 的 triggered 訊號連接）
- 無參數：此槽函式不需要接收任何參數

**第 2 行：函式本體開始**
```cpp
{
```
- 左大括號，標記函式主體的開始

**第 3 行：條件判斷**
```cpp
    if (currentFilePath.isEmpty()) {
```
- `if`：條件判斷敘述
- `currentFilePath`：QString 類型的成員變數，儲存當前正在編輯的檔案路徑
- `.isEmpty()`：QString 的成員函式，檢查字串是否為空（長度為 0）
- 此行判斷是否有已存在的檔案路徑，如果為空表示這是一個新文件，尚未儲存過

**第 4-5 行：首次儲存的處理**
```cpp
        // 如果沒有當前檔案路徑，則呼叫另存為
        onSaveAsTriggered();
```
- `// 如果沒有當前檔案路徑，則呼叫另存為`：註解說明，解釋此程式碼區塊的用途
- `onSaveAsTriggered();`：呼叫另存新檔函式，顯示檔案儲存對話框讓使用者選擇儲存位置和檔名
- 這是智慧型儲存的關鍵：首次儲存時自動轉換為「另存新檔」操作

**第 6 行：條件判斷的 else 分支**
```cpp
    } else {
```
- `else`：如果 currentFilePath 不為空（即已有檔案路徑），執行此分支

**第 7-8 行：直接儲存到現有檔案**
```cpp
        // 儲存到當前檔案
        saveToFile(currentFilePath);
```
- `// 儲存到當前檔案`：註解說明
- `saveToFile(currentFilePath);`：呼叫共用的儲存函式，直接將內容寫入到已存在的檔案路徑
- 傳入參數 `currentFilePath`：使用已記錄的檔案路徑，不需要再次詢問使用者

**第 9-10 行：函式結束**
```cpp
    }
}
```
- 第一個 `}`：結束 if-else 條件判斷區塊
- 第二個 `}`：結束整個函式定義

**執行流程總結**：
1. 函式被呼叫（通常由使用者按下 Ctrl+S 或點擊儲存按鈕觸發）
2. 檢查 `currentFilePath` 是否為空
3. 若為空 → 呼叫 `onSaveAsTriggered()` 顯示儲存對話框
4. 若不為空 → 呼叫 `saveToFile()` 直接儲存到現有檔案
5. 函式執行完畢，返回呼叫處

### 1.3 功能說明
**儲存功能**是一個智慧型的檔案保存機制，其運作邏輯如下：

1. **檢查當前檔案路徑**：
   - 程式會先檢查 `currentFilePath` 成員變數
   - 此變數儲存當前正在編輯的檔案路徑

2. **智慧判斷**：
   - **情況A - 首次儲存**：如果 `currentFilePath` 為空（表示是新建文件或尚未儲存過），則自動觸發「另存新檔」對話框
   - **情況B - 已有檔案**：如果已經有檔案路徑，則直接覆蓋儲存到該檔案

3. **使用時機**：
   - 快速儲存已開啟的文件
   - 不需要重新選擇儲存位置
   - 適合頻繁保存工作進度

### 1.4 設計優點
- **智慧化**：自動判斷是否需要顯示檔案選擇對話框
- **效率高**：已存檔的文件可快速覆蓋儲存
- **使用者友善**：符合一般文字編輯器的使用習慣

## 二、另存新檔功能 (Save As Function)

### 2.1 功能位置
- **檔案選單**：檔案(E) → 另存(A)
- **工具列**：另存按鈕（磁碟圖示變體）

### 2.2 程式碼實作
```cpp
void MainWindow::onSaveAsTriggered()
{
    QString fileName = QFileDialog::getSaveFileName(
        this,
        tr("另存檔案"),
        "",
        tr("文字檔案 (*.txt);;所有檔案 (*.*)")
    );
    
    if (!fileName.isEmpty()) {
        saveToFile(fileName);
        currentFilePath = fileName;
    }
}
```

#### 逐行程式碼說明：

**第 1 行：函式定義**
```cpp
void MainWindow::onSaveAsTriggered()
```
- `void`：函式沒有回傳值
- `MainWindow::`：這是 MainWindow 類別的成員函式
- `onSaveAsTriggered()`：函式名稱，當「另存新檔」動作被觸發時呼叫（與 action_A 的 triggered 訊號連接）
- 無參數：此槽函式不需要接收任何參數

**第 2 行：函式本體開始**
```cpp
{
```
- 左大括號，標記函式主體的開始

**第 3-7 行：開啟儲存檔案對話框**
```cpp
    QString fileName = QFileDialog::getSaveFileName(
        this,
        tr("另存檔案"),
        "",
        tr("文字檔案 (*.txt);;所有檔案 (*.*)")
    );
```
- `QString fileName =`：宣告 QString 類型變數 fileName，用來儲存使用者選擇的檔案路徑
- `QFileDialog::getSaveFileName(`：Qt 的靜態函式，顯示系統標準的儲存檔案對話框
- `this`：（第一個參數）父視窗指標，對話框將顯示在主視窗之上，並設定為模態對話框
- `tr("另存檔案")`：（第二個參數）對話框的標題文字，`tr()` 用於支援多語言翻譯
- `""`：（第三個參數）預設目錄路徑，空字串表示使用系統預設位置（通常是上次使用的目錄或使用者的文件資料夾）
- `tr("文字檔案 (*.txt);;所有檔案 (*.*)")`：（第四個參數）檔案類型篩選器
  - `文字檔案 (*.txt)`：第一個選項，只顯示 .txt 檔案
  - `;;`：分隔符號，用於分隔不同的篩選器選項
  - `所有檔案 (*.*)`：第二個選項，顯示所有檔案類型
- 如果使用者點擊「儲存」，返回所選擇的完整檔案路徑；如果點擊「取消」，返回空字串

**第 8 行：空白行**
- 增加程式碼可讀性，將對話框呼叫與後續處理邏輯分隔開

**第 9 行：檢查使用者是否選擇了檔案**
```cpp
    if (!fileName.isEmpty()) {
```
- `if`：條件判斷敘述
- `!`：邏輯 NOT 運算子，反轉條件
- `fileName.isEmpty()`：檢查 fileName 是否為空字串
- `!fileName.isEmpty()`：表示「fileName 不是空字串」，即使用者確實選擇了檔案並點擊儲存
- 此判斷用於排除使用者點擊「取消」按鈕的情況

**第 10 行：執行實際的檔案儲存**
```cpp
        saveToFile(fileName);
```
- `saveToFile(fileName);`：呼叫共用的檔案儲存函式
- 傳入參數 `fileName`：使用者在對話框中選擇的檔案路徑
- 此函式會將 textEdit 的內容寫入到指定的檔案

**第 11 行：更新當前檔案路徑**
```cpp
        currentFilePath = fileName;
```
- `currentFilePath = fileName;`：將成員變數 currentFilePath 更新為新的檔案路徑
- 這個更新非常重要：
  - 之後使用「儲存」功能（Ctrl+S）時，會直接儲存到這個新路徑
  - 不需要再次詢問使用者選擇檔案
  - 記錄當前正在編輯的檔案資訊

**第 12 行：結束 if 判斷區塊**
```cpp
    }
```
- 右大括號，結束 if 條件判斷區塊

**第 13 行：函式結束**
```cpp
}
```
- 右大括號，結束整個函式定義

**執行流程總結**：
1. 函式被呼叫（使用者點擊「檔案」→「另存(A)」選單項目）
2. 顯示系統的儲存檔案對話框
3. 使用者在對話框中選擇儲存位置、輸入檔名、選擇檔案類型
4. 如果使用者點擊「儲存」：
   - 呼叫 `saveToFile()` 將內容寫入檔案
   - 更新 `currentFilePath` 為新檔案路徑
5. 如果使用者點擊「取消」：
   - fileName 為空字串，不執行任何儲存操作
6. 函式執行完畢，返回呼叫處

### 2.3 功能說明
**另存新檔功能**提供完整的檔案儲存對話框，讓使用者可以：

1. **開啟檔案選擇對話框**：
   - 使用 Qt 的 `QFileDialog::getSaveFileName()` 方法
   - 對話框標題顯示「另存檔案」

2. **選擇儲存位置和檔名**：
   - 使用者可以瀏覽檔案系統
   - 自由輸入新的檔案名稱
   - 選擇儲存的目錄位置

3. **檔案類型篩選**：
   - 提供兩種檔案類型選項：
     - `文字檔案 (*.txt)`：預設選項，適合文字內容
     - `所有檔案 (*.*)`：允許使用任意副檔名

4. **儲存並更新路徑**：
   - 如果使用者確認儲存（沒有取消），則：
     - 呼叫 `saveToFile()` 執行實際儲存
     - 更新 `currentFilePath` 為新的檔案路徑

### 2.4 使用時機
- 將文件儲存為新檔案
- 在不同位置建立文件副本
- 以不同檔名儲存文件
- 首次儲存新建立的文件

### 2.5 設計優點
- **靈活性**：可自由選擇儲存位置和檔名
- **安全性**：不會覆蓋原始檔案（除非使用者主動選擇）
- **檔案類型支援**：支援多種檔案格式

## 三、共用儲存機制 (Shared Save Mechanism)

### 3.1 核心儲存函式
```cpp
void MainWindow::saveToFile(const QString &filePath)
{
    QFile file(filePath);
    if (!file.open(QIODevice::WriteOnly | QIODevice::Text)) {
        QMessageBox::warning(this, tr("儲存錯誤"), 
                           tr("無法儲存檔案: %1").arg(file.errorString()));
        return;
    }
    
    QTextStream out(&file);
    out << textEdit->toPlainText();
    
    statusbar->showMessage(tr("檔案已儲存: %1").arg(filePath), 3000);
}
```

#### 逐行程式碼說明：

**第 1 行：函式定義**
```cpp
void MainWindow::saveToFile(const QString &filePath)
```
- `void`：函式沒有回傳值
- `MainWindow::`：這是 MainWindow 類別的私有成員函式
- `saveToFile`：函式名稱，負責實際的檔案儲存操作
- `const QString &filePath`：函式參數
  - `const`：表示函式不會修改傳入的參數
  - `QString &`：參考傳遞（reference），避免複製整個字串，提高效能
  - `filePath`：參數名稱，代表要儲存的目標檔案路徑

**第 2 行：函式本體開始**
```cpp
{
```
- 左大括號，標記函式主體的開始

**第 3 行：建立檔案物件**
```cpp
    QFile file(filePath);
```
- `QFile`：Qt 的檔案處理類別，用於讀寫檔案
- `file`：QFile 物件的變數名稱
- `(filePath)`：建構函式參數，指定要操作的檔案路徑
- 此時只是建立物件，尚未真正開啟檔案

**第 4 行：嘗試開啟檔案並檢查是否成功**
```cpp
    if (!file.open(QIODevice::WriteOnly | QIODevice::Text)) {
```
- `if`：條件判斷敘述
- `file.open()`：開啟檔案的函式，成功返回 true，失敗返回 false
- `QIODevice::WriteOnly`：開啟模式旗標，表示「只寫」模式
  - 檔案會被清空（如果已存在）
  - 只允許寫入操作，不允許讀取
- `|`：位元 OR 運算子，用於組合多個旗標
- `QIODevice::Text`：文字模式旗標
  - 在 Windows 上會自動處理換行符號轉換（\n ↔ \r\n）
  - 確保跨平台的文字檔相容性
- `!`：邏輯 NOT 運算子，反轉條件
- 整體意思：如果檔案開啟失敗，則執行 if 區塊內的錯誤處理

**第 5-6 行：顯示錯誤訊息**
```cpp
        QMessageBox::warning(this, tr("儲存錯誤"), 
                           tr("無法儲存檔案: %1").arg(file.errorString()));
```
- `QMessageBox::warning()`：Qt 的靜態函式，顯示警告對話框
- `this`：（第一個參數）父視窗指標，對話框會顯示在主視窗之上
- `tr("儲存錯誤")`：（第二個參數）對話框標題文字，使用 tr() 支援多語言
- `tr("無法儲存檔案: %1")`：（第三個參數）錯誤訊息內容
  - `%1`：佔位符號，會被後面的參數取代
  - `.arg(file.errorString())`：取得系統錯誤訊息並填入 %1 位置
  - `file.errorString()`：返回具體的錯誤原因（例如：「權限不足」、「磁碟已滿」等）

**第 7 行：提前結束函式**
```cpp
        return;
```
- `return;`：立即結束函式執行，返回到呼叫處
- 因為檔案開啟失敗，無法繼續後續的寫入操作，必須提前退出
- 阻止程式繼續執行可能導致崩潰的程式碼

**第 8 行：結束錯誤處理區塊**
```cpp
    }
```
- 右大括號，結束 if 條件判斷區塊
- 如果程式執行到此處，表示檔案已成功開啟

**第 9 行：空白行**
- 增加程式碼可讀性，將錯誤處理與正常流程分隔開

**第 10 行：建立文字串流物件**
```cpp
    QTextStream out(&file);
```
- `QTextStream`：Qt 的文字串流類別，提供便利的文字讀寫介面
- `out`：串流物件的變數名稱（output 的縮寫）
- `(&file)`：建構函式參數，將串流連接到 file 物件
  - `&`：取得 file 的記憶體位址（指標）
  - 串流會將資料寫入到這個檔案物件
- 使用 QTextStream 的優點：自動處理字串編碼、緩衝管理等細節

**第 11 行：寫入文字內容到檔案**
```cpp
    out << textEdit->toPlainText();
```
- `out <<`：串流輸出運算子，將資料寫入串流
- `textEdit`：QTextEdit 控件的指標，代表文字編輯器
- `->toPlainText()`：QTextEdit 的成員函式，取得編輯器中的純文字內容
  - 返回 QString 類型
  - 不包含任何格式化資訊（粗體、斜體等）
  - 包含所有換行和空白字元
- 整行作用：將文字編輯器的內容全部寫入到檔案中

**第 12 行：空白行**
- 增加程式碼可讀性，將檔案操作與 UI 回饋分隔開

**第 13 行：顯示成功訊息**
```cpp
    statusbar->showMessage(tr("檔案已儲存: %1").arg(filePath), 3000);
```
- `statusbar`：指向狀態列控件的指標（QStatusBar）
- `->showMessage()`：在狀態列顯示訊息的函式
- `tr("檔案已儲存: %1")`：要顯示的訊息文字
  - `%1`：佔位符號，會被檔案路徑取代
  - `.arg(filePath)`：將 filePath 參數填入 %1 位置
- `3000`：（第二個參數）訊息顯示時間，單位為毫秒（3000 毫秒 = 3 秒）
  - 3 秒後訊息會自動消失
  - 讓使用者知道檔案已成功儲存，並顯示儲存的位置

**第 14 行：函式結束**
```cpp
}
```
- 右大括號，結束整個函式定義
- 函式執行完畢時：
  - QFile 物件 `file` 自動關閉（解構函式會處理）
  - QTextStream 物件 `out` 自動清理
  - 所有資料已安全寫入磁碟

**執行流程總結**：
1. 函式被呼叫，接收目標檔案路徑參數
2. 建立 QFile 物件，關聯到指定的檔案路徑
3. 嘗試以「只寫+文字模式」開啟檔案
4. 如果開啟失敗：
   - 顯示錯誤對話框，告知使用者具體錯誤原因
   - 提前結束函式
5. 如果開啟成功：
   - 建立 QTextStream 串流物件
   - 從 textEdit 取得純文字內容
   - 將內容寫入檔案
   - 在狀態列顯示成功訊息（顯示 3 秒）
6. 函式結束，自動關閉檔案

**重要特性**：
- **錯誤處理**：檔案操作失敗時會給予使用者明確的錯誤訊息
- **資源管理**：檔案會自動關閉，不會造成資源洩漏
- **使用者回饋**：透過狀態列即時告知操作結果
- **跨平台**：使用 QIODevice::Text 確保在不同作業系統上都能正確處理文字檔

### 3.2 功能說明
**saveToFile()** 是 Save 和 Save As 共用的底層儲存函式：

1. **開啟檔案**：
   - 創建 `QFile` 物件
   - 使用 `QIODevice::WriteOnly` 模式（只寫）
   - 使用 `QIODevice::Text` 模式（文字模式）

2. **錯誤處理**：
   - 如果無法開啟檔案（權限不足、路徑不存在等）
   - 顯示警告對話框，說明錯誤原因
   - 使用 `file.errorString()` 取得系統錯誤訊息

3. **寫入內容**：
   - 使用 `QTextStream` 進行文字串流寫入
   - 從 `textEdit` 控件取得純文字內容（`toPlainText()`）
   - 將內容寫入檔案

4. **使用者回饋**：
   - 在狀態列顯示成功訊息
   - 顯示實際儲存的檔案路徑
   - 訊息顯示 3 秒後自動消失

### 3.3 設計優點

### 5.2 訊號與槽連接（mainwindow.cpp）
```cpp
// 連接 Save 和 Save As 動作
connect(action_S, &QAction::triggered, this, &MainWindow::onSaveTriggered);
connect(action_A, &QAction::triggered, this, &MainWindow::onSaveAsTriggered);
```

**說明**：
- `action_S` 觸發時，呼叫 `onSaveTriggered()` 槽函式
- `action_A` 觸發時，呼叫 `onSaveAsTriggered()` 槽函式
- 使用 Qt 的訊號槽機制實現事件處理

#### 逐行程式碼說明：

**第 1 行：連接儲存動作的訊號與槽**
```cpp
connect(action_S, &QAction::triggered, this, &MainWindow::onSaveTriggered);
```
- `connect()`：Qt 的訊號槽連接函式，用來建立物件之間的通訊機制
- `action_S`：儲存動作物件（在 UI 檔案中定義），代表「儲存(S)」選單項目和工具列按鈕
- `&QAction::triggered`：訊號的函式指標，當使用者點擊選單或按鈕時會發出此訊號
- `this`：接收訊號的物件，指向當前的 MainWindow 實例
- `&MainWindow::onSaveTriggered`：槽函式的函式指標，訊號發出時要執行的成員函式

**第 2 行：連接另存新檔動作的訊號與槽**
```cpp
connect(action_A, &QAction::triggered, this, &MainWindow::onSaveAsTriggered);
```
- `connect()`：同上，Qt 的訊號槽連接函式
- `action_A`：另存新檔動作物件（在 UI 檔案中定義），代表「另存(A)」選單項目和工具列按鈕
- `&QAction::triggered`：當使用者觸發此動作時發出的訊號
- `this`：接收訊號的物件，指向當前的 MainWindow 實例
- `&MainWindow::onSaveAsTriggered`：處理另存新檔的槽函式

**執行時機**：
這兩行程式碼在 MainWindow 的建構函式中執行，確保在視窗顯示之前就已經建立好訊號與槽的連接關係。

## 九、使用流程範例

### 範例 1：首次儲存新文件
1. 使用者在 `textEdit` 輸入內容
2. 按下 `Ctrl+S` 或點擊儲存按鈕
3. 因為 `currentFilePath` 為空，自動觸發另存新檔對話框
4. 使用者選擇儲存位置和檔名（例如：`C:\Documents\my_file.txt`）
5. 程式將文字內容寫入檔案
6. `currentFilePath` 更新為 `C:\Documents\my_file.txt`
7. 狀態列顯示「檔案已儲存: C:\Documents\my_file.txt」

### 範例 2：編輯現有文件後儲存
1. 使用者修改已開啟的文件內容
2. 按下 `Ctrl+S`
3. 因為 `currentFilePath` 已有值，直接覆蓋儲存
4. 無需重新選擇檔案
5. 狀態列顯示確認訊息

### 範例 3：另存為新檔案
1. 使用者編輯文件後想保留原始檔案
2. 點擊「檔案」→「另存(A)」
3. 在對話框中輸入新檔名（例如：`my_file_v2.txt`）
4. 程式將內容儲存到新檔案
5. `currentFilePath` 更新為新檔案路徑
6. 之後的 Save 操作將儲存到新檔案
